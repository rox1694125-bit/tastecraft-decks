# 2026-06-08 Project Log

## 本轮目标

Review the first six generated single-slide tax-comparison images and identify why they failed the user's taste bar.

## 用户反馈

- All six generated images were rejected.
- The outputs were too simple and looked like ordinary card-based charts.
- If the result is only a plain chart layout, HTML or normal PPT editing is faster.
- The value of image generation should be the ability to create visuals beyond ordinary HTML/PPT layouts.
- Future defaults should target 4K when the generation path supports explicit size control.

## 关键发现

- The first prompt set over-constrained the model into plain "two panels plus donut charts" output.
- The templates were treated mostly as color palettes instead of complete art-direction systems.
- Negative prompts such as "no decorative clutter" removed too much visual richness.
- The prompts lacked role framing, premium visual ambition, information hierarchy, material depth, background graphics, and explicit color usage rules.
- Dense Chinese tax labels remain a text-accuracy risk even when the visual layout is acceptable.

## 分辨率记录

- The six built-in `image_gen` outputs were `1672x941`, not 2K or 4K.
- Local imagegen fallback documentation says `gpt-image-2` CLI/API workflows can request `3840x2160` for 4K landscape output.
- Built-in `image_gen` does not expose the same explicit size controls in the current Codex tool surface.

## 下一轮方向

- Rewrite prompts as high-end one-page visual design briefs, not chart layout instructions.
- Add "visual ambition" to the acceptance rubric.
- Use 4K CLI/API fallback for final-quality tests when `OPENAI_API_KEY` and network access are available.
- Consider a hybrid workflow when exact Chinese text matters: AI generates premium visual assets/background, deterministic HTML/PPT rendering places exact charts and text, final output is flattened as a 4K image.

## Follow-up A/B: AI Background + Exact Overlay

The user approved testing the next version: pure AI final page versus AI-generated premium visual system/background/assets plus deterministic HTML overlay.

Approach C:

- Generate an AI background/visual-system asset with no text and no charts.
- Overlay exact Chinese text and deterministic charts in HTML/CSS.
- Export the final composition through Chrome headless at `3840x2160`.
- Local files:
  - `outputs/ab-tax-2026-06-08/ai-visual-system-background.png`
  - `outputs/ab-tax-2026-06-08/ai-bg-overlay-tax-comparison.png`

Findings:

- Prompting the built-in `image_gen` tool with "4K-quality" did not change the actual output dimensions; the background asset was still `1672x941`.
- The final overlay export is true `3840x2160` because the HTML renderer controls the canvas size.
- Approach C keeps some AI visual atmosphere from Approach A while preserving exact text and numbers like Approach B.
- User feedback after reviewing Approach C: the effect is too weak. The correct project path is pure AI whole-slide finished images, not AI background/visual system plus HTML precise overlay.

Final route decision:

- Pure AI finished-page generation is the V1 main path.
- Hybrid overlay is no longer a default or recommended path for this project stage.
- The prompt system should improve pure AI visual ambition, composition, material depth, and page-level art direction instead of trying to recover quality with deterministic overlay.
- Hybrid overlay may remain as a fallback when exact text/data is more important than visual ceiling, but it should not drive the core skill design.

Resolution rule:

- Prompt words like "4K" can improve perceived sharpness or style intent, but cannot guarantee actual pixel dimensions when the tool surface does not expose a size parameter.
- Actual 4K generation requires a size-controlled path such as CLI/API `gpt-image-2 --size 3840x2160`, or a deterministic renderer such as HTML/PPT export at `3840x2160`.

## Gemini Nano Banana Pro Pricing Check

The user asked for the API price of `gemini-3-pro-image-preview / Nano Banana Pro` with `aspectRatio=16:9` and `imageSize=4K`.

Official Google pricing currently maps Nano Banana Pro to Gemini 3 Pro Image pricing:

- Free tier: not available for Gemini 3 Pro Image.
- Standard paid tier image output: `$120.00 / 1,000,000 image output tokens`.
- 1K/2K output image: `$0.134` per image.
- 4K output image: `$0.24` per image.
- Batch or Flex 4K output image: `$0.12` per image.
- Input text/image cost is separate, but for normal prompt-only slide generation the dominant cost is the output image.

Cost implication for TasteCraft testing:

- 6-template single-slide horizontal review at 4K Standard: about `$1.44` plus small input-token cost.
- 6 templates with up to 3 attempts each: about `$4.32` plus small input-token cost.
- If Batch/Flex is acceptable, image output cost is roughly half, but latency and availability tradeoffs need separate testing.

User routing decision:

- Use Standard for normal interactive testing.
- Use Batch for larger batch generation runs.
- Do not use Priority for TasteCraft testing.
- Flex is not the default unless the user explicitly wants lower cost while preserving synchronous-style calls.

OAuth clarification:

- Google AI Pro / Google One provides expanded usage for Google consumer and product surfaces such as Gemini, AI Studio, Flow, and related Google apps.
- The Gemini API can authenticate with OAuth, but OAuth is an authentication method tied to a Google Cloud project and enabled API. It does not automatically convert Google AI Pro consumer subscription limits into free Gemini API image-generation quota.
- For automated generation from Codex, the reliable route remains Gemini API with a Google Cloud project that has usable paid Gemini API quota, using Standard for normal work and Batch for bulk runs.
