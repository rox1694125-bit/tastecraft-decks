# Synthetic Demo Gallery

This gallery contains text-only fictional examples. It does not contain binary images, generated slide files, real logos, real screenshots, customer data, or private brand assets.

Each entry demonstrates how a TasteCraft deck brief might describe audience, format, density, visual direction, and QA concerns before an execution skill builds the deck.

## Entry 1: Fictional Harbor Foods

Synthetic status: fully fictional.

Brief:

- Audience: board members reviewing a quarterly operating plan.
- Goal: explain margin pressure, channel mix, and recovery actions.
- Tone: direct, calm, operational.
- Target format: editable PPTX.
- Slide count: 12.
- Density: reading-first for data pages, balanced for chapter pages.

Theme notes:

- Palette: off-white background, charcoal text, muted green primary, amber accent.
- Logo policy: no logo, text-only fictional name on cover and closing.
- Imagery: none required; use charts and operating diagrams.
- Forbidden elements: real grocery brands, real packaging, real supplier names.

QA focus:

- Make every data slide answer one board-level question.
- Keep charts editable.
- Use invented metrics only.
- Add clear source note: `Synthetic example data`.

## Entry 2: Northstar Demo Fund

Synthetic status: fully fictional.

Brief:

- Audience: seed investors evaluating a climate analytics concept.
- Goal: show problem, model, traction proxy, and capital plan.
- Tone: ambitious but evidence-led.
- Target format: image-enhanced PPTX.
- Slide count: 10.
- Density: speaker-led for cover and vision pages, balanced for traction pages.

Theme notes:

- Palette: deep graphite, bright cyan accent, pale mint secondary.
- Logo policy: no real logo; use a text wordmark only if needed.
- Imagery: abstract infrastructure grids, fictional dashboards, non-identifying data flows.
- Forbidden elements: real fund logos, real investor names, recognizable city landmarks, real satellite imagery.

Prompt-pack intent:

- Cover visual: abstract climate-risk grid over generic terrain, no map labels.
- Method visual: layered model diagram with invented variable names.
- Closing visual: non-branded horizon scene, no real infrastructure.

QA focus:

- Confirm every image prompt before export.
- Keep generated visuals supportive, not proof-like.
- Clearly mark all traction metrics as synthetic.

## Entry 3: Example Robotics Lab

Synthetic status: fully fictional.

Brief:

- Audience: enterprise buyers comparing automation pilots.
- Goal: demonstrate workflow states and implementation risks.
- Tone: precise, technical, pragmatic.
- Target format: HTML deck.
- Slide count: 14.
- Density: balanced.

Theme notes:

- Palette: light neutral background, slate text, cobalt primary, orange warning accent.
- Logo policy: no logo.
- Imagery: synthetic UI wireframes and system diagrams only.
- Forbidden elements: real robot designs, real customer facilities, real vendor logos.

HTML interaction ideas:

- Slide with clickable pilot phases.
- Before/after process map.
- Risk matrix with keyboard-accessible tabs.

QA focus:

- Check mobile and desktop layout.
- Avoid overlapping text in compact views.
- Ensure interactive elements have visible focus states.

## Entry 4: Synthetic City Transit

Synthetic status: fully fictional.

Brief:

- Audience: operations supervisors in a training workshop.
- Goal: teach incident escalation and passenger communication flow.
- Tone: clear, procedural, bilingual labels where useful.
- Target format: editable PPTX.
- Slide count: 18.
- Density: appendix-heavy for reference slides, balanced for workshop slides.

Theme notes:

- Palette: white background, dark ink text, blue primary, yellow caution accent.
- Logo policy: no logo.
- Imagery: simple route diagrams with invented station names.
- Forbidden elements: real transit maps, agency marks, real incident reports.

QA focus:

- Keep procedures scan-friendly.
- Use fictional station names.
- Avoid implying official transit guidance.
- Provide role-based summary pages.
