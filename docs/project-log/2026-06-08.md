# 2026-06-08 Project Log

## 本轮目标

Review the first six generated single-slide tax-comparison images and identify why they failed the user's taste bar.

## 用户反馈

- All six generated images were rejected.
- The outputs were too simple and looked like ordinary card-based charts.
- If the result is only a plain chart layout, HTML or normal PPT editing is faster.
- The value of image generation should be the ability to create visuals beyond ordinary HTML/PPT layouts.
- Future defaults should target 4K when the generation path supports explicit size control.

## 关键发现

- The first prompt set over-constrained the model into plain "two panels plus donut charts" output.
- The templates were treated mostly as color palettes instead of complete art-direction systems.
- Negative prompts such as "no decorative clutter" removed too much visual richness.
- The prompts lacked role framing, premium visual ambition, information hierarchy, material depth, background graphics, and explicit color usage rules.
- Dense Chinese tax labels remain a text-accuracy risk even when the visual layout is acceptable.

## 分辨率记录

- The six built-in `image_gen` outputs were `1672x941`, not 2K or 4K.
- Local imagegen fallback documentation says `gpt-image-2` CLI/API workflows can request `3840x2160` for 4K landscape output.
- Built-in `image_gen` does not expose the same explicit size controls in the current Codex tool surface.

## 下一轮方向

- Rewrite prompts as high-end one-page visual design briefs, not chart layout instructions.
- Add "visual ambition" to the acceptance rubric.
- Use 4K CLI/API fallback for final-quality tests when `OPENAI_API_KEY` and network access are available.
- Consider a hybrid workflow when exact Chinese text matters: AI generates premium visual assets/background, deterministic HTML/PPT rendering places exact charts and text, final output is flattened as a 4K image.

## A/B Test: Pure AI vs Hybrid

The user asked to compare two approaches instead of assuming the hybrid workflow is better.

Approach A:

- Pure built-in image generation, one prompt, whole-slide image with text and charts.
- Output copied locally as `outputs/ab-tax-2026-06-08/pure-ai-tax-comparison.png`.
- Actual dimensions: `1672x941`.
- Strength: stronger visual atmosphere, premium navy/gold tone, symbolic imagery, more obvious image-generation value.
- Weakness: text is not fully controllable; the model changed wording and inserted extra symbolic elements.

Approach B:

- Deterministic HTML/CSS composition exported by Chrome headless.
- Output saved locally as `outputs/ab-tax-2026-06-08/hybrid-tax-comparison.png`.
- Actual dimensions: `3840x2160`.
- Strength: exact Chinese text, exact numbers, deterministic charts, true 4K output.
- Weakness: still reads more like a high-end HTML/PPT infographic than an image-model-native visual.

Current conclusion:

- Pure AI has a higher visual ceiling but lower text/data reliability.
- Hybrid has higher production reliability but needs richer art direction or AI-generated background/assets to avoid looking like polished HTML.
- The next useful test is not "plain HTML vs pure AI"; it is "pure AI final page" vs "AI-generated premium visual system + deterministic text/chart overlay".
