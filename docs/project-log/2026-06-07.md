# 2026-06-07 Project Log

## 本轮目标

Define how to test TasteCraft prompt templates against real generated PPT images.

## 本轮决策

- Start with single-slide comparison instead of full-deck generation.
- Use one realistic slide brief and test it across all six style templates.
- Simulate the final user workflow: user provides slide content, TasteCraft drafts prompts, Codex generates images, user reviews, then feedback is distilled into prompt improvements.
- Keep every completed feedback round documented and pushed to GitHub.
- Whole-slide image generation may include text, but key text mistakes are rejects.

## 场景差异沉淀

- Investor roadshow: premium, credible, growth-oriented, moderate density.
- Internal report: practical, structured, action-oriented, readable.
- Keynote or launch: bold, emotional, low-density, strong first impression.
- Board review: restrained, decision-supporting, evidence-oriented.
- Product demo: workflow context, product value, clear user benefit.
- Technical brief: precise hierarchy, mechanism clarity, low decoration.
- Training: friendly, repeatable, easy to follow.

## 下一步

- Ask the user for one single-slide test brief.
- Generate one image candidate per template.
- Capture feedback with `docs/prompt-testing-protocol.md`.
- Distill reusable prompt changes and push the round to GitHub.
