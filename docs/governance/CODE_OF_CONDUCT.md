# Code Of Conduct

TasteCraft Decks contributors are expected to keep discussion professional, specific, and useful.

## Expected Behavior

- Assume shared intent to improve the project.
- Critique ideas, code, docs, and public claims with concrete evidence.
- Keep feedback actionable.
- Respect different levels of familiarity with presentation design, open-source process, and agent workflows.
- Avoid publishing private or identifying information.

## Unacceptable Behavior

- Harassment, threats, or discriminatory language.
- Sharing private decks, customer details, or non-public brand material.
- Impersonating a real organization in synthetic examples.
- Repeatedly derailing technical review after maintainers redirect the discussion.
- Pressuring contributors to bypass data, brand, or release checks.

## Reporting

Open a private maintainer channel if one exists for the project host. If not, open a minimal issue that says a conduct review is needed, without including private details in the public issue.

Maintainers should review reports promptly and may remove content, close issues, block contributors, or update policy when needed.
