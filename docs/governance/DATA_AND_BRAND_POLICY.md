# Data And Brand Policy

TasteCraft Decks is designed for public open-source collaboration. Public content must be safe to inspect, fork, package, and redistribute.

## Public Repository Rules

Allowed:

- fictional organizations;
- synthetic metrics;
- neutral placeholder marks described in text only;
- generated prompt descriptions;
- markdown examples;
- schemas and validation fixtures with invented values.

Not allowed:

- real customer decks;
- real company logos;
- private brand books;
- screenshots from client systems;
- unreleased product screenshots;
- personal data;
- files from `knowledge/brand-kits/_local/`;
- binary demo assets unless a future release policy explicitly approves them.

## Brand-Kit Handling

The public repo may contain synthetic brand-kit examples. Real brand kits must remain local-only and should not be referenced in public demo docs.

When writing public examples, use names that make fiction obvious, such as:

- `Fictional Harbor Foods`;
- `Example Robotics Lab`;
- `Northstar Demo Fund`;
- `Synthetic City Transit`.

## AI Image Policy

Public prompt examples should not ask an image model to recreate a real logo, real product packaging, real venue, real person, or protected brand style. Prompt packs must include forbidden elements when image generation is enabled.

## Review Rule

If reviewers cannot quickly tell whether an example is synthetic, revise it until they can.
