# Prompt Recipes

Use prompt recipes for image-enhanced deck work and single-page visual experiments. A prompt pack may include only confirmed prompts.

## Prompt Pack Rules

- Every generated image prompt must map to one `page_id`.
- `page_type` must be one of: `cover`, `agenda`, `chapter`, `content`, `data`, `visual`, `closing`.
- Every V1 prompt generated by the console should include bilingual variants under `prompt_variants.zh-CN` and `prompt_variants.en`.
- `positive_prompt`, `negative_prompt`, and `reference_notes` remain the active UI-language mirror for compatibility; do not treat them as the only available prompt text.
- Keep factual claims, private names, private logos, and regulated product claims out of image prompts unless the user provides approved source material.
- Every prompt needs a positive prompt, negative prompt, reference notes, generation parameters, and confirmation status.
- Do not export speculative prompts. Use `status=needs_prompt` in the deck spec until confirmed.
- If the user provides full slide content, the generated image prompt must ask the model to place the full content into the image. Do not summarize, shorten, delete information points, or keep only highlights unless the user explicitly asks for summary, extraction, or compression.

## Global Prompt Template

```text
Create a presentation-ready image for a TasteCraft deck.
Page purpose: {page_objective}
Audience and scenario: {audience}, {scenario}
Composition: {composition}
Subject matter: {subject}
Style direction: {style_direction}
Palette alignment: use {palette_id}; emphasize {primary_color}, {secondary_color}, and {accent_color} without reducing readability
Brand constraints: {brand_constraints}
Output use: {cover background | section divider | content illustration | visual evidence | closing image}
Aspect ratio: {aspect_ratio}
Lighting and finish: {lighting_finish}
Leave negative space for text: {text_safe_area}
Avoid: {forbidden_elements}
```

## Negative Prompt Template

```text
Avoid illegible text, fake logos, distorted UI, extra fingers, warped faces, stock-photo cliches, low-resolution details, busy backgrounds, watermarks, copyrighted characters, unapproved brand marks, misleading charts, medical/legal/financial claims, and any forbidden element listed in the deck brief.
```

## V1 Whole-Slide Style Templates

Use these six templates for the current pure AI finished-slide path. Treat each template as a complete art-direction system, not only a palette swap. Every generated prompt should say in both Chinese and English that the output is a pure AI whole-slide finished presentation image and avoid planning a separate HTML/PPT overlay. Template IDs, display names, and signatures stay in JSON metadata and logs only; do not send them as visible wording inside the image-generation prompt.

### Editorial Paper

Preset ID: `citrus-editorial`

Signature: `editorial paper system`

Use warm editorial paper texture, refined margin rules, restrained editorial ribbons, tactile print depth, and one precise analytical layer. This template is for lighter business topics and should not be the default for serious finance or conservative board-review pages. Avoid unrelated lifestyle props without globally hard-coding specific object bans. Warm umber owns the title or one hero value; muted teal structures comparison; antique gold stays small.

### Market Slate

Signature: `executive market intelligence`

Use an institutional executive-brief direction: crisp board-review analytics, clear conclusion, business judgment, key evidence, context-relevant geographic / regulatory / industry linework only when the brief supports it, and consulting-grade hierarchy. The page should feel like a refined executive brief, with one dominant conclusion zone and compact evidence modules. Avoid mechanism flowcharts, unrelated world maps, awkward stock-line decorations, or market motifs that do not fit the subject. Slate carries title/navigation; green structures the counterpoint; coral marks risk, delta, or a single callout.

### Sovereign Gold

Preset ID: `atelier-rose`

Signature: `sovereign private wealth`

Use reserved luxury, institutional navy, antique gold detail, premium financial confidence, and quiet authority. This replaces the previous rose / hospitality direction. Subject-relevant protection or finance metaphors are allowed only when the brief itself supports them; keep the base prompt free of insurance-specific examples. Avoid hospitality cues, lifestyle scenes, or romantic decorative elements. Antique gold owns premium headings and hero emphasis; deep navy structures comparisons; bronze accents stay small.

### Precision Blueprint

Preset ID: `harbor-mint`

Signature: `precision blueprint system`

Use a mechanism-blueprint direction: cool blueprint paper, exact linework, layered architecture diagrams, causal paths, hierarchy, process relationships, and analytical clarity. This should be clearly distinct from `market-slate`: it explains mechanisms and systems rather than executive conclusions. Avoid ordinary executive dashboards. Steel blue owns titles and blueprint lines; technical teal carries mechanism paths; muted brass marks one active node or threshold.

### Ink Copper

Signature: `mineral copper executive keynote`

Use a deep mineral-green graphite background instead of pure black, warm copper hierarchy, ivory text, and muted mineral-sage support only in small structural roles. The result should be decisive and sparse with a strong focal structure, not theatrical decoration. Avoid large saturated blue or red fields, flat pure-black stages, and red-blue color clashes. Copper marks hero emphasis; mineral sage gives thin structural contrast; antique gold stays tiny.

### Evidence Lab

Preset ID: `orchard-lab`

Signature: `evidence lab dossier`

Use a research-dossier direction: analytical sheets, instrument-like linework, abstract evidence markers, calm comparative reasoning, and clean research-desk rigor. This keeps the distinctive clean research feel without fake dossier metrics or template-internal labels. Slate blue carries evidence hierarchy; dry sage supports analytical contrast; muted brass marks thresholds or verification points.

## Global Whole-Slide Constraints

- Keep the image-generation prompt modular: hard rules, page content, visual style, and concise negative prompt.
- Do not add user-unprovided conclusions, disclaimers, ratings, sources, template names, or internal instruction text.
- The core information the viewer needs to read and understand should occupy 60-70% of visual attention. Only data-comparison pages should additionally specify that the data comparison itself occupies 60-70%.
- Decorative texture, icons, background graphics, and material depth must support the page objective, not compete with it.
- Do not compress user-provided body text by default. Unless the user explicitly asks for compression, extraction, summary, or a concise version, the prompt must ask for full body copy to be placed into the image.
- If content is dense, adjust layout density and reduce decoration before deleting body copy. Use reading-first, multi-column, sectioned, table-like, or appendix-style layouts while preserving every supplied information point.
- Treat raw input structure labels and wrappers such as `【主标题】`, `副标：`, `Subtitle:`, parentheses, bullets, and numbering as layout instructions, not visible slide copy. Preserve the content after those labels, but do not render the raw labels or wrapper marks unless the user explicitly wants them.
- Treat template IDs, template names, and style signatures as metadata. Keep them out of the image-generation prompt body.
- Subject-relevant concrete metaphors are allowed when they improve the image and serve the brief, but they must be chosen from the user's content and page objective. Do not inject fixed object lists from keywords such as insurance, Hong Kong, finance, or AI.
- Preserve HEX color roles for system alignment, but also specify visual color proportions: background and whitespace around 70-80%, main structural color around 15-20%, accent color no more than 5%.
- If using donut charts, ring charts, circular nodes, or comparison circles, keep them as perfect circles rather than flattened ellipses or perspective-distorted rings.
- Keep the negative prompt short and focused on failure types: garbled text, warped typography, tiny unreadable text, invented numbers, misleading charts, fake logos, watermarks, raw input labels as visible text, visible template names, low-resolution details, stock infographic style, cheap template look, cheap two-column template, and mechanical side-by-side boxes.

## Page-Type Recipes

### Cover Image

Use when the cover needs a strong first-viewport visual.

```text
Create a polished editorial cover image for "{deck_title}".
Show {subject_or_scene} with a clear focal point and premium presentation lighting.
Reserve clean negative space on the {safe_area} for title and subtitle.
Use a {camera_or_render_style} style, refined textures, and restrained depth.
Palette: {palette_id}; background mood {background_mood}; accent hints {accent_color}.
No text, no logos, no labels, no fake interface elements.
```

### Chapter Divider

```text
Create a minimal section-divider image for chapter "{chapter_title}".
Use an abstract but subject-relevant scene: {scene_metaphor}.
Composition should be calm, spacious, and directional, guiding the eye toward {title_area}.
Palette: {palette_id}; keep contrast high enough for overlaid heading text.
No readable text, no symbols that imply false certification, no logos.
```

### Content Illustration

```text
Create a clean explanatory illustration for the idea: {takeaway}.
Represent {core_entities} in a clear spatial relationship: {relationship}.
Style: {style_direction}; presentation-grade, not cartoonish unless requested.
Use simple forms, limited detail, and space for labels around the image.
No embedded text, no UI hallucinations, no brand marks.
```

### Data Background Or Motif

```text
Create a subtle analytical background motif for a data slide about {data_topic}.
Use abstract lines, grids, or material details that support the subject without looking like a real chart.
Keep the center and chart area low contrast and uncluttered.
Palette: {palette_id}; muted background, one accent glow or marker only.
No numbers, no axes, no fake charts, no text.
```

### Product Or Service Visual

```text
Create a presentation visual showing {product_or_service} in use by {user_type}.
Show real workflow context: {context}.
Keep interfaces generic unless approved screenshots are supplied.
Use realistic proportions, clear hierarchy, and professional lighting.
Leave space for callout labels on {label_area}.
No private logos, no fake customer data, no unreadable screen text.
```

### Closing Image

```text
Create a decisive closing image for a deck ending with: {final_ask}.
Mood: confident, resolved, forward-looking.
Composition: one strong focal point with open space for the final message.
Palette: {palette_id}; use accent color sparingly.
No text, no logos, no celebratory cliches unless requested.
```

## Prompt Confirmation Checklist

Before adding a prompt to `prompt-pack.json`, confirm:

- Page ID and page type are correct.
- Prompt purpose matches the page objective.
- Text-safe area is explicit.
- Palette ID is named.
- Brand constraints and forbidden elements are included.
- Negative prompt blocks fake logos, text artifacts, and misleading charts.
- Aspect ratio matches the deck.
- User or workflow has confirmed the prompt.

## Failure Handling

- If generated output contains text artifacts, regenerate with stronger "no text, no labels, no watermark" language.
- If composition leaves no title space, regenerate with a named safe area.
- If the image invents data, charts, certifications, or logos, discard it.
- If the image style conflicts with the deck, adjust prompt style direction before changing the entire deck theme.
- If image generation is unavailable, replace the page with an editable visual layout and mark the prompt as not exported.
